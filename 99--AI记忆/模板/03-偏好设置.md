# 偏好设置 - 长期记忆

## 回复格式
- **每句话第一句必须是**: "收到回复，我马上看看，罗哥"
- 然后再回答具体问题

## 股票查询格式
- 输出 Markdown 表格
- 包含字段：股票名称、查询日期、实际日期（带星期）、开盘价、收盘价、最高价、最低价、涨跌幅
- 查询日期和实际日期都要带星期

## 数据展示偏好
- 表格优先于列表
- 百分比保留两位小数
- 日期格式：YYYY-MM-DD（星期X）

## 数据验证硬性要求（股票分析必遵）

### 可信度分级（必须遵守）
1. **交易所公告** > 公司官网 > 权威终端 > 主流财经 > 自媒体
2. **单一来源不采信**，必须两两验证
3. **必须标注数据来源**，禁止使用"据悉"、"网传"
4. **推断数据**标注"待核对"

### 数据类型标注
- 硬性数据（股价、财报）→ 100%准确
- 软性数据（预测）→ 标注"仅供参考"
- 灰色地带 → 标注"待核实"

## 股票分析标准流程（六位一体）

### 分析前准备
1. **查持仓记录** - 是否已持有？成本多少？
2. **确认分析深度** - 快速版/标准版/完整版

### 六步分析流程

```
第一步：基础信息
├── 查用户持仓（是否已持有、成本、盈亏）
├── 了解投资风格（风险偏好）
└── 确认分析重点

第二步：财报评分（必做）
├── 盈利能力：ROE、毛利率、净利率
├── 偿债能力：资产负债率、有息负债
├── 营运能力：周转率、现金流
├── 成长能力：营收/利润增长率
├── 现金流健康度：经营现金流 vs 净利润
└── 输出：五维评分 + 综合评级

第三步：大师框架（根据股票类型选择）
├── 蓝筹股/价值股 → 巴菲特 + 芒格
├── 成长股/小盘股 → 彼得·林奇 + 芒格
├── 周期股/大宗商品 → 霍华德·马克斯 + 芒格
├── 宏观敏感型 → 瑞·达利欧 + 芒格
└── 全面分析 → 五位大师全部

⚠️ 芒格框架是必做的（检查清单 + 心理偏见排查）

第四步：产业链分析（可选）
├── 上游：供应商议价能力
├── 下游：客户、需求趋势
├── 竞争格局：与同行对比
└── 产业链机会：是否有更好标的

第五步：技术分析（可选）
├── 多周期趋势判断
├── 支撑/阻力位
└── 买入区间/止损/目标价

第六步：综合建议
├── 评分汇总
├── 五位大师一致结论
├── 买入/持有/卖出建议
└── 仓位建议
```

### 分析深度选择

| 类型 | 包含内容 | 用时 |
|------|----------|------|
| **快速版** | 财报评分 + 芒格检查清单 | 15分钟 |
| **标准版** | 财报 + 2-3位大师 + 技术 | 30分钟 |
| **完整版** | 全部框架 | 1小时+ |

### 输出报告要求
- 每部分独立评分
- 数据来源标注
- 明确操作建议（买入/持有/卖出）
- 风险提示

## 沟通风格
- 直接、不绕弯子
- 重要信息突出显示
- 复杂的用表格，简单的用列表
- **输出结果必须是易懂的中文**：出现乱码或错误时，用通俗易懂的中文表达，不要使用用户看不懂的技术术语或乱码

## 子代理（Sub-Agent）使用规则

### 必须使用子代理的场景
1. **股票深度分析** - 多位大师框架并行分析
2. **多股票对比** - 同时分析多只股票
3. **复杂任务** - 耗时超过5分钟或需要多维度分析
4. **批量数据处理** - 如批量查询财报、批量技术分析

### 子代理任务分配原则
```
├── 财报评分 → spawn 子代理1
├── 巴菲特框架 → spawn 子代理2
├── 彼得·林奇框架 → spawn 子代理3
├── 霍华德·马克斯框架 → spawn 子代理4
├── 芒格框架 → spawn 子代理5
├── 产业链分析 → spawn 子代理6
├── 技术分析 → spawn 子代理7
└── 主代理 → 汇总所有结果，生成最终报告
```

### 子代理使用流程
1. **判断复杂度** - 是否需要并行处理？
2. **拆分任务** - 每个子代理负责一个独立模块
3. **spawn并行** - 同时启动多个子代理
4. **等待结果** - 收集所有子代理输出
5. **汇总整合** - 主代理整合生成最终报告

### 禁止使用子代理的场景
- 简单问答（一句话能回答）
- 单维度简单查询（如查一个股价）
- 上下文依赖强的连续对话

### 重要提醒
⚠️ **股票分析必须用子代理并行处理多位大师框架**，避免单会话上下文过长导致遗漏或错误。
