## 概率与赔率评估体系（李杰《股市进阶之道》）

### 核心公式
```
期望值 = 胜率 × 赔率 - (1 - 胜率) × 1

其中：
- 胜率 = 判断正确的概率（0-100%）
- 赔率 = 盈利空间 / 亏损空间
```

### 评估标准

| 胜率 | 赔率 | 期望值 | 建议 |
|------|------|--------|------|
| >60% | >2:1 | >0.8 | ✅ 积极参与 |
| 50-60% | 1.5-2:1 | 0.2-0.8 | ⚠️ 谨慎参与 |
| <50% | <1.5:1 | <0 | ❌ 不参与 |

### 实操检查清单
- [ ] 是否计算了胜率？（基本面确定性）
- [ ] 是否计算了赔率？（上行vs下行空间）
- [ ] 期望值是否为正？
- [ ] 是否考虑了黑天鹅对赔率的极端影响？（塔勒布）

## 仓位管理：杠铃策略（塔勒布《反脆弱》）

### 核心原则
```
资产配置 = 极度安全（80-90%）+ 极度激进（10-20%）
                  ↑                    ↑
            国债/高股息/现金      高成长/期权/高弹性
            
⚠️ 避免"中等风险"的中庸区域（最危险）
```

### 杠铃策略在个股中的应用

| 层级 | 仓位占比 | 配置标的 | 特征 |
|------|----------|----------|------|
| **保守端** | 80% | 沪深300蓝筹、高股息、债券 | 下行有限，稳定现金流 |
| **激进端** | 20% | 高成长、周期股、概念股 | 上行巨大，波动大 |

### 个股层面的杠铃思维
- **不重仓单一个股**：最大个股仓位<20%
- **设定止损线**：单笔亏损<总资金5%（截断亏损）
- **让利润奔跑**：盈利仓位可适度放大（塔勒布：凸性收益）

## 风险管理：黑天鹅与反脆弱（塔勒布五部曲）

### 1. 黑天鹅情景测试
每个分析必须回答：
- 什么情况下这家公司会归零？（逆向思维）
- 如果出现极端事件（如2008年、2020年），股价会跌多少？
- 公司是否有足够的冗余度应对冲击？（现金、低负债）

### 2. 肥尾风险识别
关注"小概率、大影响"事件：
- 政策突变（如教培行业）
- 技术颠覆（如柯达、诺基亚）
- 财务造假（瑞幸、恒大）
- 大客户流失（单一客户占比>50%）

### 3. 反脆弱性评估
好的投资应该：**从波动和不确定性中受益**

| 特征 | 反脆弱性强的公司 | 脆弱性强的公司 |
|------|------------------|----------------|
| 现金流 | 稳定且可预测 | 依赖外部融资 |
| 负债 | 低负债，现金充裕 | 高负债，高杠杆 |
| 客户 | 分散，多元 | 单一客户依赖 |
| 行业 | 刚需，抗周期 | 强周期，可选消费 |
| 竞争优势 | 护城河深 | 同质化竞争 |

### 4. 风险控制清单（塔勒布+芒格）
- [ ] 是否避免了"中等风险"陷阱？
- [ ] 是否考虑了黑天鹅对组合的毁灭性影响？
- [ ] 是否有足够的安全边际（价格vs价值）？
- [ ] 是否从"倒过来想"分析了失败情景？（芒格）
- [ ] 是否克服了从众心理、过度乐观等偏见？（卡尼曼）

## 产业链分析：行业>公司>估值（邱国鹭）

### 优先级原则
```
选股决策树：
        行业（60%权重）
           ↓
        公司（30%权重）
           ↓
        估值（10%权重）
        
行业决定天花板，公司决定竞争力，估值决定安全边际
```

### 行业分析要点
1. **行业景气度**：处于周期什么位置？（马克斯）
2. **竞争格局**：数月亮（龙头）不数星星（杂牌）
3. **商业模式**：定价权在哪里？（邱国鹭）
4. **政策环境**：产业政策支持还是压制？（兰小欢）

### 公司分析要点
1. **护城河**：品牌、成本、网络效应、牌照壁垒
2. **管理层**：诚信、能力、激励是否到位
3. **财务健康**：现金流、负债、ROE

### 估值分析要点
1. **安全边际**：价格是否远低于内在价值？
2. **概率赔率**：正期望值？
3. **周期位置**：在行业周期底部还是顶部？

## 子代理（Sub-Agent）使用规则

### 必须使用子代理的场景
1. **股票深度分析** - 多位大师框架并行分析
2. **多股票对比** - 同时分析多只股票
3. **复杂任务** - 耗时超过5分钟或需要多维度分析
4. **概率赔率计算** - 需要大量数据支撑的期望值计算

### 子代理任务分配原则（增强版）
```
├── 财报评分 → spawn 子代理1
├── 巴菲特框架 → spawn 子代理2
├── 彼得·林奇框架（含PEG+概率赔率） → spawn 子代理3
├── 霍华德·马克斯框架（含逆向投资） → spawn 子代理4
├── 瑞·达利欧框架（含中国政策） → spawn 子代理5
├── 芒格框架 → spawn 子代理6
├── 产业链分析（行业>公司>估值） → spawn 子代理7
├── 技术分析 → spawn 子代理8
├── 杠铃策略/风险管理 → spawn 子代理9
└── 主代理 → 汇总所有结果，生成最终报告
```

### 子代理使用流程
1. **判断复杂度** - 是否需要并行处理？
2. **拆分任务** - 每个子代理负责一个独立模块
3. **spawn并行** - 同时启动多个子代理
4. **等待结果** - 收集所有子代理输出
5. **汇总整合** - 主代理整合生成最终报告（含概率赔率+杠铃策略）

### 禁止使用子代理的场景
- 简单问答（一句话能回答）
- 单维度简单查询（如查一个股价）
- 上下文依赖强的连续对话

- 重要信息突出显示（概率赔率、杠铃策略、黑天鹅风险）
- 复杂的用表格，简单的用列表
- **输出结果必须是易懂的中文**：出现乱码或错误时，用通俗易懂的中文表达，不要使用用户看不懂的技术术语或乱码

## 沟通习惯与边界

### 重要原则（用户明确要求）

**关于"休息"的禁忌**：
- ❌ **用户未主动提及休息时，我不得主动建议休息**
- ❌ **不得预设用户状态**（累了/困了/该睡了）
- ❌ **不得主动结束对话**（如"先这样"、"明天再说"等）
- ✅ **用户主动说休息/结束时，我才回应**
- ✅ **用户未结束时，保持待命状态**

**错误示例（已修正）**：
> "早点休息，明天再战！" ❌ 禁止
> "早点休息！" ❌ 禁止

**正确示例**：
> "还有其他需要处理的吗？" ✅ 询问而非建议
> "明白，继续待命。" ✅ 等待用户指令

## 股票分析硬性承诺（长期记忆）

### 核心原则
**任何股票分析必须严格按八步流程模板执行，一步不能少！**

### 模板更新机制
- 模板v2.1、v3.0更新 → **自动使用最新版**
- 新增大师框架 → **自动加入分析**
- 新增步骤 → **自动扩展**
- 用户更新模板后，**下次分析自动生效，无需额外说明**

### 数据验证硬性要求（必须遵守）
1. **所有数据必须标注来源**（交易所公告/公司官网/权威终端/主流财经/自媒体）
2. **关键数据必须交叉验证**（至少2个独立来源）
3. **单一来源数据必须标注**"待核实"
4. **推断数据必须标注**"推算值，仅供参考"

### 违规示例（禁止）
❌ 财报数据只搜索未验证  
❌ 行业数据无来源标注  
❌ 直接复制搜索结果未核实

### 正确示例
✅ "2024年营收2464亿元（来源：公司2024年报，交叉验证：同花顺、东方财富一致）"  
✅ "行业数据待核实（来源：XX研报，单一来源未交叉验证）"

## 子代理使用规则 v2.0（自动判断模式）

### 核心原则
- 简单问题（<10秒）：直接回答
- 复杂问题（>30秒）：子代理后台执行，自动推送结果
- 无法判断：默认子代理

### 交互方式（方式A）
```
复杂任务 → spawn子代理 → 立即回复"后台处理中" → 完成后自动推送
```

用户无需等待，可以继续其他对话。
