## 文件发送功能使用手册（2026-03-13 更新）

### 功能说明
通过 Kimi 消息通道直接发送文件给用户（支持邮件/聊天窗口）。

### 使用方法

**工具**: `message`  
**Action**: `send`  
**关键参数**:

```json
{
  "action": "send",
  "filePath": "/绝对路径/到/文件.md",
  "filename": "显示给用户的文件名.md",
  "mimeType": "text/markdown",
  "message": "邮件正文说明"
}
```

### 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| `filePath` | ✅ | 文件的绝对路径（如 `/root/.openclaw/workspace/...`） |
| `filename` | ❌ | 用户看到的文件名（不传则使用原文件名） |
| `mimeType` | ❌ | 文件类型（markdown用 `text/markdown`，txt用 `text/plain`） |
| `message` | ❌ | 邮件/消息正文内容 |

### 成功案例（2026-03-13）

发送 `06-财报评分框架.md` 到用户邮箱：

```json
{
  "action": "send",
  "filePath": "/root/.openclaw/workspace/99--AI记忆/模板/06-财报评分框架.md",
  "filename": "06-财报评分框架_v2.0.md",
  "mimeType": "text/markdown",
  "message": "罗哥，财报评分框架 v2.0 已生成完成！..."
}
```

**结果**: ✅ 成功送达，文件显示在聊天窗口

### 常见失败原因（2026-03-12 教训）

| 失败方式 | 原因 | 正确做法 |
|----------|------|----------|
| ❌ 用 `exec` + `curl` 间接发送 | 绕过官方通道，参数复杂易错 | ✅ 直接用 `message` 工具的 `filePath` 参数 |
| ❌ 文件路径错误或权限不足 | 系统找不到文件 | ✅ 使用绝对路径，确认文件存在且可读 |
| ❌ 未指定 `mimeType` | 文件类型识别错误 | ✅ 明确指定 `text/markdown` 等类型 |
| ❌ 使用相对路径 | 工作目录不确定导致路径错误 | ✅ 一律使用绝对路径 |

### 核心口诀
> 官方通道最可靠，`message` + `filePath` + `mimeType`，绝对路径保平安。
